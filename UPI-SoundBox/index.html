Audio Protocol

Audible (default for ESP32): standard tones — more reliable through physical speakers at distance.
Silent (ultrasonic): 18–22 kHz, inaudible. Use when broadcasting directly from a phone browser.

How it works

The UPI ID is encoded as FSK audio tones using ggwave. A compressed UPI ID (~10 chars) fits in a single fast packet.

For ESP32 deployment: download upi.wav from the broadcast screen, place it in data/ next to the sketch, run LittleFS Data Upload, then flash the sketch. No SD card needed — audio lives in ESP32 internal flash.

For phone-to-phone: open this page on the merchant phone, tap Broadcast. Customer opens the same page, taps Listen.
